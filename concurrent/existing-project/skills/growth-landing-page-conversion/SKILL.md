# Growth Landing Page Conversion

## Purpose
Increase conversion quality by structuring landing pages around audience intent and friction removal.

## Use When
- Building a new marketing landing page.
- Optimizing existing pages with weak conversion.
- Aligning messaging with product capabilities.

## Inputs Required
- `specs/requirements.md`
- `specs/ui-spec.md`
- `docs/README.md`
- Target persona, offer, and conversion goal

## Steps
1. Define primary conversion event and secondary events.
2. Draft message hierarchy: promise -> proof -> mechanism -> CTA.
3. Identify objection handling blocks and trust elements.
4. Specify CTA placement and intent-matched variants.
5. Define experiment variants with success/failure thresholds.

## Output Contract
- Landing strategy section in `specs/growth-plan.md`
- Hand-offable copy and UX requirements in `handoffs/growth-to-documentation.md`

## Safety and Security
- Avoid deceptive claims or dark-pattern recommendations.
- Ensure claims are supportable by product behavior and evidence.
